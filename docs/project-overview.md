# 프로젝트 개요

상태: decision

## 프로젝트 이름

HSA AI

HSA는 Hanyang Support Agent의 약자다.

## 한 줄 설명

HSA AI는 백엔드가 전달한 쇼핑몰 고객 문의와 맥락을 바탕으로, 운영자가 사용할 수 있는 답변 초안을 반환하는 AI 보조 서비스다.

## 전체 시스템 연결 방향

HSA 서비스의 기본 연결 방향은 다음과 같다.

```text
Frontend <-> Backend <-> AI
```

역할 구분은 다음과 같다.

| 영역 | 책임 |
| --- | --- |
| Frontend | 관리자 화면 표시, 사용자의 검토/수정/전송 액션 입력 |
| Backend | 문의 저장, 주문/배송 DB 조회, AI 호출, 응답 전송, 로그 저장, 프론트엔드 API 제공 |
| AI | 정책 문서 관리, 백엔드가 전달한 문의와 운영 데이터 맥락 기반 답변 초안 반환 |

AI는 프론트엔드나 고객 채널과 직접 통신하지 않는다.

## 해결하려는 문제

1인 쇼핑몰이나 신생 쇼핑몰 운영자는 반복적인 고객 문의를 직접 확인하고 답변해야 한다.

특히 다음 문의가 자주 반복된다.

- 배송 상태 문의
- 송장번호 문의
- 교환/환불 문의
- 상품 정보 문의
- 정책 확인이 필요한 문의

HSA AI는 백엔드가 전달한 문의와 운영 데이터 맥락, 그리고 AI 파트가 관리하는 정책 문서를 바탕으로 답변 초안을 작성하여 운영자의 응대를 보조한다.

## AI 파트가 담당하는 것

초기 PoC에서 AI 파트는 다음 책임을 가진다.

1. 백엔드가 전달한 문의와 운영 데이터 맥락 기반 답변 초안 생성
2. AI 파트가 관리하는 Markdown 정책 문서와 LlamaIndex 검색 결과 활용
3. 답변 생성에 사용한 근거 또는 이유 반환
4. 관리자 검토 필요 여부 판단
5. 위험 태그와 처리 상태 같은 최소 메타데이터 반환
6. 평가와 로그에 필요한 결과 필드 반환

## AI 파트가 담당하지 않는 것

AI 파트는 다음 책임을 직접 소유하지 않는다.

- 프론트엔드와 직접 통신
- 문의 채널 연동
- 고객 문의 원문 수집
- 문의 저장과 상태 관리
- 관리자 화면 구현
- 고객 응답 실제 전송
- 주문/배송 DB 조회 또는 직접 소유
- 정책 문서의 백엔드 저장/검색
- 백엔드 DB schema 최종 확정
- 배포 인프라 최종 확정

## 초기 PoC 범위

초기 PoC는 전체 고객센터 자동화가 아니라, 백엔드가 전달한 고객 문의 1건에 대해 답변 초안과 판단 정보를 안정적으로 반환하는 흐름을 검증한다.

처리 흐름은 다음과 같다.

```text
Frontend에서 관리자/사용자 액션 발생
-> Backend가 문의 원문과 주문/배송 등 운영 데이터 맥락을 저장/조회
-> Backend가 AI에 답변 초안 생성을 요청
-> AI가 Markdown 정책 문서를 LlamaIndex로 검색
-> AI가 답변 초안, 검토 필요 여부, 판단 이유를 반환
-> Backend가 결과를 저장하고 Frontend에 전달
-> Frontend에서 관리자가 확인/수정
-> Backend가 고객 응답 전송
```

## 초기 PoC에서 하지 않는 것

- AI가 고객에게 직접 응답을 전송하지 않는다.
- AI가 프론트엔드와 직접 통신하지 않는다.
- 환불, 교환, 클레임에 대한 최종 판단을 AI가 하지 않는다.
- 근거 문서가 없는 답변을 생성하지 않는다.

## 성공 기준

초기 PoC는 다음을 만족하면 성공으로 본다.

- 백엔드가 화면에 표시할 수 있는 답변 초안을 반환한다.
- 정책 문서는 AI 파트에서 관리한다.
- 관리자 검토가 필요한 문의를 명확히 표시한다.
- AI 결과가 자연어뿐 아니라 구조화된 JSON 형태로 반환된다.
- 백엔드가 저장하고 화면에 표시할 수 있는 필드를 반환한다.
- 근거 없는 답변 생성을 피하는 정책이 포함된다.
